Hello JSP World!

HTML 형식에서 자바코드를 사용할 수 있는 "자바 언어"로 서블릿에서 JAVA 코드로 응답 HTML을 구현했던 복잡함을 간단하게 작성, 해결할 수 있다.
--> JSP의 가장 큰 장점은 Servlet에서 비지니스 로직(java)와 프리젠테이션 로직(html)을 분리할 수 있다는 것이다.

JSP Elements 표현법

1. JSP 스크립팅 원소(Scripting Element)

JSP 페이지에 자바 코드를 직접 기술할 수 있기 해주는 기능이다.

  1. 선언문 : <%! 자바코드 %> 멤버 변수와 메소드를 선언하기 위해 사용

  2. 스크립틀릿 : <% 자바코드 %> JSP에서 자바코드를 사용하기 위해 사용

  3. 표현식(출력식) : <%= 자바코드 %> 자바에서 작성한 값을 출력하기 위해 사용

2. 지시어(Directive)

JSP page 전체에 영향을 미치는 정보를 기술할 때 쓰임

  1. page : 현재의 JSP 페이지를 컨테이너에서 처리하는데 필요한 각종 속성을 기술하는 부분으로 보통 소스의 맨 앞에 위치
    • language : 사용할 스크립트 언어 유형을 지정
    • contentType : 웹 브라우저가 받아 볼 페이지의 형식, 인코딩 방식 지정
    • pageEncoding : JSP 파일에 기록 된 자바 코드의 인코딩 방식 지정
    • import : 자바의 import와 같은 의미
    • errorPage / isErrorPage : 오류 페이지 관리

  2. include : jsp 파일이 자바 파일로 바뀔 때 자바 문서에 삽입 되어 컴파일 되는 파일 추가

  3. taglib : JSP 기능을 좀 더 확장할 수 있는 인자로 정의 된 사용자 정의 태그를 가져와
    사용할 수 있는 방법 제공 (EX.JSTL 등 -> 나중에 배움)
